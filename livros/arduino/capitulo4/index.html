    Programando com Arduino: Primeiros Passos e Exemplos Práticos

    4.1. Introdução à Programação Arduino

    A programação do Arduino é realizada utilizando uma linguagem baseada em C/C++, simplificada para facilitar o aprendizado e a utilização por iniciantes. O ambiente de desenvolvimento (IDE) fornece uma estrutura básica para os programas, chamados de "sketches". Um sketch Arduino possui duas funções principais: setup() e loop().

    • setup(): Esta função é executada apenas uma vez, quando o Arduino é ligado ou resetado. É utilizada para inicializar configurações, como definir o modo dos pinos (entrada ou saída), iniciar a comunicação serial, etc.
    • loop(): Esta função é executada continuamente em um ciclo infinito, após a função setup() ser concluída. É onde a lógica principal do programa é implementada: ler dados de sensores, tomar decisões e controlar atuadores.

    4.2. Estrutura Básica de um Sketch

    Todo sketch Arduino segue uma estrutura básica:

    void setup() {
  // Inicialização das configurações
}

void loop() {
  // Lógica principal do programa
}

    4.3 Constantes e Variáveis

    Constantes e variáveis são elementos fundamentais na programação Arduino, permitindo que você armazene e manipule dados dentro de seus sketches.

    Constantes: São valores predefinidos que não podem ser alterados durante a execução do programa. Elas representam valores fixos e imutáveis. No Arduino, existem três grupos principais de constantes:

    • TRUE / FALSE: Constantes booleanas que definem estados lógicos. TRUE é representado por qualquer valor diferente de zero (geralmente 1), e FALSE é representado por zero (0). São utilizadas em expressões condicionais e lógicas. 
      bool ledLigado = TRUE; // Declara uma variável booleana e atribui o valor TRUE
if (ledLigado) {
  digitalWrite(ledPin, HIGH); // Liga o LED se a variável for TRUE
}
    • HIGH / LOW: Definem os níveis de tensão nos pinos digitais. HIGH geralmente representa 5V (ou 3.3V em algumas placas), e LOW representa 0V (GND). São utilizados para definir o estado de um pino digital como ligado ou desligado. 
      digitalWrite(ledPin, HIGH); // Define o pino do LED como HIGH (5V), ligando o LED
    • INPUT / OUTPUT: Definem se um pino digital será configurado como uma entrada (para ler dados de um sensor, por exemplo) ou como uma saída (para controlar um LED, por exemplo). São utilizados na função pinMode(). 
      pinMode(ledPin, OUTPUT);   // Define o pino do LED como uma saída
pinMode(buttonPin, INPUT); // Define o pino do botão como uma entrada

    Variáveis: São posições nomeadas na memória do Arduino, utilizadas para armazenar informações que podem ser alteradas durante a execução do programa. Cada variável possui um nome (um identificador) e um tipo (que define o tipo de dado que pode ser armazenado na variável).

    Escopo: O escopo de uma variável define onde ela pode ser acessada e utilizada dentro do código.

    • Variáveis globais: São declaradas fora de qualquer função (geralmente no início do sketch) e podem ser acessadas e modificadas por qualquer função no programa. 
      int ledPin = 13; // Variável global: pode ser usada em setup() e loop()

void setup() {
  pinMode(ledPin, OUTPUT);
}

void loop() {
  digitalWrite(ledPin, HIGH);
}
    • Variáveis locais: São declaradas dentro de uma função e só podem ser acessadas e modificadas dentro dessa função. 
      void loop() {
  int sensorValue = analogRead(A0); // Variável local: só pode ser usada dentro de loop()
  // ...
}

    Declaração: Antes de usar uma variável, é necessário declará-la, especificando seu tipo e seu nome. A sintaxe geral para declarar uma variável é:

    tipo nomeDaVariavel;

    Tipos de Variáveis: O Arduino suporta diversos tipos de variáveis, cada um adequado para armazenar diferentes tipos de dados. Alguns dos tipos mais comuns são:

    • byte: Armazena um valor inteiro de 8 bits, variando de 0 a 255. Útil para armazenar pequenos valores inteiros ou dados binários. 
      byte data = 120; // Armazena o valor 120 em um byte
    • int: Armazena um valor inteiro de 16 bits. A faixa de valores depende da placa Arduino utilizada:
      • Arduino Uno: -32,768 a 32,767
      • Arduino Due: -2,147,483,648 a 2,147,483,647 Útil para armazenar valores inteiros dentro dessa faixa. 
        int temperature = 25; // Armazena a temperatura em graus Celsius
    • long: Armazena um valor inteiro de 32 bits, variando de -2,147,483,648 a 2,147,483,647. Útil para armazenar valores inteiros grandes. 
      long distance = 150000; // Armazena a distância em milímetros
    • float: Armazena um número de ponto flutuante (número com casas decimais) de 32 bits. Útil para armazenar valores com precisão decimal. 
      float pi = 3.14159; // Armazena o valor de pi
    • boolean: Armazena um valor booleano, que pode ser TRUE ou FALSE. Útil para representar estados lógicos. 
      boolean buttonPressed = false; // Armazena o estado do botão (pressionado ou não)
    • char: Armazena um único caractere (letra, número, símbolo) de 8 bits, de acordo com a tabela ASCII. 
      char initial = 'J'; // Armazena a inicial do nome

    Escolher o tipo de variável correto é importante para otimizar o uso da memória do Arduino e garantir que os dados sejam armazenados com precisão.

    4.3. Primeiros Passos: Blink (Piscando um LED)

    Um dos primeiros exemplos que todo iniciante em Arduino experimenta é o "Blink", que consiste em fazer um LED piscar. Para isso, siga os passos:

    1. Conecte um LED ao pino digital 13 do Arduino, utilizando um resistor de 220 ohms em série para limitar a corrente e proteger o LED. O pino 13 já possui um resistor embutido na placa, mas é uma boa prática usar um resistor externo.
    2. Abra o IDE Arduino e crie um novo sketch.
    3. Copie e cole o seguinte código:
    // Define o pino do LED
int ledPin = 13;

void setup() {
  // Define o pino do LED como saída
  pinMode(ledPin, OUTPUT);
}

void loop() {
  // Liga o LED
  digitalWrite(ledPin, HIGH);
  // Espera 1 segundo
  delay(1000);
  // Desliga o LED
  digitalWrite(ledPin, LOW);
  // Espera 1 segundo
  delay(1000);
}
    1. Explicação do código:
      • int ledPin = 13;: Define uma variável inteira chamada ledPin e atribui o valor 13 a ela. Essa variável representa o número do pino digital ao qual o LED está conectado.
      • pinMode(ledPin, OUTPUT);: Configura o pino definido na variável ledPin como uma saída. Isso significa que o Arduino pode enviar sinais elétricos através desse pino para controlar o LED.
      • digitalWrite(ledPin, HIGH);: Envia um sinal HIGH (5V) para o pino do LED, ligando-o.
      • delay(1000);: Pausa a execução do programa por 1000 milissegundos (1 segundo).
      • digitalWrite(ledPin, LOW);: Envia um sinal LOW (0V) para o pino do LED, desligando-o.
    2. Conecte o Arduino ao computador através do cabo USB.
    3. Selecione a placa Arduino correta no menu "Ferramentas > Placa".
    4. Selecione a porta serial correta no menu "Ferramentas > Porta".
    5. Clique no botão "Carregar" (seta para a direita) para enviar o código para o Arduino.

    Após o upload, o LED conectado ao pino 13 deverá começar a piscar a cada segundo.

    4.4. Lendo Entradas Digitais: Botão

    Este exemplo demonstra como ler o estado de um botão conectado ao Arduino.

    1. Conecte um botão ao pino digital 2 do Arduino. Utilize um resistor de 10k ohms como pull-down, conectando-o entre o pino do botão e o GND. Conecte o outro terminal do botão ao 5V.
    2. Copie e cole o seguinte código:
    // Define o pino do botão
int buttonPin = 2;
// Define o pino do LED
int ledPin = 13;

void setup() {
  // Define o pino do botão como entrada
  pinMode(buttonPin, INPUT);
  // Define o pino do LED como saída
  pinMode(ledPin, OUTPUT);
}

void loop() {
  // Lê o estado do botão
  int buttonState = digitalRead(buttonPin);

  // Se o botão estiver pressionado
  if (buttonState == HIGH) {
    // Liga o LED
    digitalWrite(ledPin, HIGH);
  } else {
    // Desliga o LED
    digitalWrite(ledPin, LOW);
  }
}
    1. Explicação do código:
      • int buttonPin = 2;: Define uma variável inteira chamada buttonPin e atribui o valor 2 a ela. Essa variável representa o número do pino digital ao qual o botão está conectado.
      • int ledPin = 13;: Define uma variável inteira chamada ledPin e atribui o valor 13 a ela. Essa variável representa o número do pino digital ao qual o LED está conectado.
      • pinMode(buttonPin, INPUT);: Configura o pino definido na variável buttonPin como uma entrada. Isso significa que o Arduino irá ler o estado elétrico desse pino.
      • pinMode(ledPin, OUTPUT);: Configura o pino definido na variável ledPin como uma saída.
      • int buttonState = digitalRead(buttonPin);: Lê o estado digital do pino do botão (HIGH ou LOW) e armazena o valor na variável buttonState.
      • if (buttonState == HIGH) { ... } else { ... }: Uma estrutura condicional que verifica se o botão está pressionado (HIGH). Se estiver, o LED é ligado. Caso contrário, o LED é desligado.
    2. Carregue o código para o Arduino.

    Agora, ao pressionar o botão, o LED deverá acender. Ao soltar o botão, o LED deverá apagar.

    4.5. Lendo Entradas Analógicas: Potenciômetro

    Este exemplo demonstra como ler um valor analógico de um potenciômetro e usar esse valor para controlar o brilho de um LED.

    1. Conecte um potenciômetro à placa Arduino. Ligue um terminal externo do potenciômetro ao 5V, o outro terminal externo ao GND e o terminal central (o wiper) ao pino analógico A0.
    2. Conecte um LED ao pino digital 9 do Arduino, utilizando um resistor de 220 ohms em série. O pino 9 é um pino PWM, o que significa que ele pode gerar sinais PWM para controlar o brilho do LED.
    3. Copie e cole o seguinte código:
    // Define o pino do potenciômetro
int potPin = A0;
// Define o pino do LED
int ledPin = 9;

void setup() {
  // Define o pino do LED como saída
  pinMode(ledPin, OUTPUT);
}

void loop() {
  // Lê o valor do potenciômetro (0-1023)
  int potValue = analogRead(potPin);
  // Mapeia o valor do potenciômetro para a faixa de PWM (0-255)
  int ledBrightness = map(potValue, 0, 1023, 0, 255);
  // Define o brilho do LED
  analogWrite(ledPin, ledBrightness);
}
    1. Explicação do código:
      • int potPin = A0;: Define uma variável inteira chamada potPin e atribui o valor A0 a ela. Essa variável representa o número do pino analógico ao qual o potenciômetro está conectado.
      • int ledPin = 9;: Define uma variável inteira chamada ledPin e atribui o valor 9 a ela. Essa variável representa o número do pino PWM ao qual o LED está conectado.
      • pinMode(ledPin, OUTPUT);: Configura o pino definido na variável ledPin como uma saída.
      • int potValue = analogRead(potPin);: Lê o valor analógico do pino do potenciômetro (um valor entre 0 e 1023) e armazena o valor na variável potValue.
      • int ledBrightness = map(potValue, 0, 1023, 0, 255);: Mapeia o valor do potenciômetro (que varia de 0 a 1023) para a faixa de valores PWM (que varia de 0 a 255). A função map() realiza essa conversão linear.
      • analogWrite(ledPin, ledBrightness);: Envia um sinal PWM para o pino do LED, definindo o seu brilho de acordo com o valor da variável ledBrightness.
    2. Carregue o código para o Arduino.

    Agora, ao girar o potenciômetro, o brilho do LED deverá variar.

    4.6. Comunicação Serial: Enviando Dados para o Computador

    Este exemplo demonstra como enviar dados do Arduino para o computador através da comunicação serial.

    1. Não é necessário nenhum circuito externo para este exemplo, pois a comunicação serial é feita através do cabo USB.
    2. Copie e cole o seguinte código:
    void setup() {
  // Inicia a comunicação serial com uma taxa de 9600 bauds
  Serial.begin(9600);
}

void loop() {
  // Envia a mensagem "Olá, mundo!" para o computador
  Serial.println("Olá, mundo!");
  // Espera 1 segundo
  delay(1000);
}
    1. Explicação do código:
      • Serial.begin(9600);: Inicializa a comunicação serial com uma taxa de 9600 bauds (bits por segundo). Essa taxa deve ser a mesma utilizada no Monitor Serial do IDE Arduino.
      • Serial.println("Olá, mundo!");: Envia a mensagem "Olá, mundo!" para o computador através da porta serial. A função println() adiciona um caractere de nova linha ao final da mensagem.
    2. Carregue o código para o Arduino.
    3. Abra o Monitor Serial no IDE Arduino (Ferramentas > Monitor Serial).

    Você deverá ver a mensagem "Olá, mundo!" sendo exibida no Monitor Serial a cada segundo.

    4.7. Lógica e Aritmética

    Operações Aritméticas

    O Arduino suporta as operações aritméticas básicas:

    • Adição: + (ex: x = x + 1;)
    • Subtração: - (ex: x = x - 2;)
    • Multiplicação: * (ex: x = x * 3;)
    • Divisão: / (ex: x = x / 4;)
    • Módulo (Resto da divisão): % (ex: x = x % 5;)

    Símbolos Compostos

    Combinam operadores aritméticos com o operador de atribuição =, oferecendo uma forma concisa de modificar o valor de uma variável:

    • Incremento: x++ (Equivalente a x = x + 1;)
    • Decremento: x-- (Equivalente a x = x - 1;)
    • Adição e Atribuição: x += y (Equivalente a x = x + y;)
    • Subtração e Atribuição: x -= y (Equivalente a x = x - y;)
    • Multiplicação e Atribuição: x *= y (Equivalente a x = x * y;)
    • Divisão e Atribuição: x /= y (Equivalente a x = x / y;)

    Operadores de Comparação

    Utilizados para comparar dois valores. O resultado da comparação é um valor booleano (TRUE ou FALSE):

    • Igual a: == (ex: x == y)
    • Diferente de: != (ex: x != y)
    • Menor que: < (ex: x < y)
    • Maior que: > (ex: x > y)
    • Menor ou igual a: <= (ex: x <= y)
    • Maior ou igual a: >= (ex: x >= y)

    Operadores Lógicos

    Utilizados para combinar ou negar expressões booleanas:

    • AND (E lógico): && (Retorna TRUE se ambas as expressões forem TRUE) 
      if (x > 0 && x < 10) {
  // Executa se x for maior que 0 E menor que 10
}
    • OR (OU lógico): || (Retorna TRUE se pelo menos uma das expressões for TRUE) 
      if (x == 0 || y == 0) {
  // Executa se x for igual a 0 OU y for igual a 0
}
    • NOT (NÃO lógico): ! (Inverte o valor da expressão. Se a expressão for TRUE, !expressão será FALSE, e vice-versa) 
      if (!digitalRead(buttonPin)) {
  // Executa se o botão NÃO estiver pressionado (digitalRead retorna LOW)
}

    Funções Matemáticas e de Tempo

    Estas funções são essenciais para controlar o tempo e realizar cálculos matemáticos em seus projetos Arduino.

    Função Exemplo Notas
    delay(ms) delay(1000); (pausa por 1 segundo) Pausa o programa por ms milissegundos. Durante o delay, o Arduino não executa outras tarefas, mas as interrupções de hardware ainda podem ser processadas (embora o delay() possa afetar a precisão das interrupções). Use com moderação em aplicações que exigem alta responsividade.
    delayMicroseconds(us) delayMicroseconds(1000); (pausa por 1 ms) Pausa o programa por us microssegundos. Similar a delay(), mas em uma escala de tempo menor. Útil para temporizações precisas, mas também impede a execução de outras tarefas.
    millis() long startTime = millis(); if (millis() - startTime > 5000) { ... } Retorna o número de milissegundos desde que o Arduino iniciou o programa atual. É útil para medir intervalos de tempo e executar tarefas periodicamente sem bloquear o programa. A variável retorna ao zero após aproximadamente 50 dias.
    micros() unsigned long timer_start = micros(); Retorna o número de microssegundos desde o início do programa. Este número irá retornar ao zero (overflow) após aproximadamente 70 minutos. Em algumas placas Arduino, esta função tem uma resolução de alguns microssegundos.
    random(min, max) int valor = random(100, 400); Gera números pseudoaleatórios entre min (inclusivo) e max (exclusivo). Se você usar randomSeed() no início do seu programa, você pode obter diferentes sequências de números aleatórios a cada vez que o Arduino é ligado.
    randomSeed(seed) randomSeed(analogRead(0)); Define o valor inicial (seed) para o gerador de números aleatórios. Usar uma entrada analógica não conectada como seed pode fornecer um valor mais imprevisível.
    abs(x) float valor = abs(-3.14); (valor será 3.14) Retorna o valor absoluto de x. Funciona com int, float e outros tipos numéricos.
    constrain(x, a, b) sensorValue = constrain(sensorValue, 10, 100); Restringe um valor x entre os limites a e b. Se x for menor que a, retorna a. Se x for maior que b, retorna b. Caso contrário, retorna x.
    map(valor, min1, max1, min2, max2) int ledBrightness = map(analogRead(A0), 0, 1023, 0, 255); Converte valor de uma faixa (min1 a max1) para outra faixa (min2 a max2). Pode ser usado para converter a leitura de um sensor para uma faixa de valores utilizável para controlar um atuador. Pode reverter faixas (ex: 1-100 para 100-1).

    Funções para Controle de Fluxo

    Estas funções são essenciais para ditar a sequência em que as instruções são executadas dentro de um programa Arduino. Elas capacitam a criação de lógicas complexas e a tomada de decisões baseadas em condições específicas.

    A estrutura de controle if permite a execução condicional de um bloco de código. A sintaxe básica consiste em avaliar uma expressão; caso essa expressão seja considerada verdadeira (ou seja, avaliada como um valor diferente de zero), o bloco de código associado será executado. Por exemplo:

    int temperature = 26;
if (temperature > 25) {
  digitalWrite(fanPin, HIGH); // Liga a ventoinha se a temperatura for maior que 25 graus
}

    A construção if...else oferece uma bifurcação na execução do código. Se a expressão condicional for verdadeira, o primeiro bloco de código será executado; caso contrário, o bloco de código subsequente ao else será executado. Essa estrutura garante que precisamente um dos dois blocos de código seja executado. Observe o exemplo:

    int buttonState = digitalRead(buttonPin);
if (buttonState == HIGH) {
  digitalWrite(ledPin, HIGH); // Liga o LED se o botão estiver pressionado
} else {
  digitalWrite(ledPin, LOW);  // Desliga o LED se o botão não estiver pressionado
}

    A estrutura if...else if...else amplia a capacidade de testar múltiplas condições em ordem sequencial. Cada else if introduz uma nova condição a ser avaliada somente se as condições precedentes forem falsas. O bloco else final, sendo opcional, é executado apenas quando nenhuma das condições anteriores for satisfeita. Essa construção é valiosa para implementar lógicas de decisão mais complexas, como demonstrado no exemplo abaixo:

    int temperature = 31;
if (temperature > 30) {
  digitalWrite(fanPin, HIGH); // Liga a ventoinha se a temperatura for maior que 30 graus
} else if (temperature > 25) {
  digitalWrite(fanPin, LOW);  // Desliga a ventoinha se a temperatura for maior que 25 graus, mas não maior que 30
} else {
  delay(1000); // Espera 1 segundo se a temperatura for menor ou igual a 25 graus
}

    A estrutura switch...case oferece uma alternativa eficiente para comparar uma variável (ou expressão que resulte em um valor inteiro) com uma lista de valores predefinidos. Cada valor possível é representado por uma cláusula case. Quando uma correspondência é encontrada, o bloco de código associado ao case correspondente é executado. A instrução break é crucial para evitar a execução dos case subsequentes. A cláusula default, opcional, é executada se nenhum dos case corresponder ao valor da variável. Veja o exemplo:

    int sensorValue = 2;
switch (sensorValue) {
  case 1:
    digitalWrite(led1, HIGH);
    break;
  case 2:
    digitalWrite(led2, HIGH);
    break;
  default:
    digitalWrite(led1, LOW);
    digitalWrite(led2, LOW);
}

    As estruturas de repetição, como while e do...while, permitem a execução repetida de um bloco de código enquanto uma condição específica for mantida verdadeira. No caso do while, a condição é avaliada antes de cada execução do bloco de código. É imperativo garantir que a condição eventualmente se torne falsa, a fim de evitar um loop infinito. Observe o exemplo:

    int temperature = 27;
while (temperature > 25) {
  digitalWrite(coolingSystem, HIGH); // Liga o sistema de resfriamento
  temperature = readTemperature();  // Lê a nova temperatura
}
digitalWrite(coolingSystem, LOW); // Desliga o sistema de resfriamento quando a temperatura for menor ou igual a 25

    A estrutura do...while é semelhante ao while, com a distinção de que o bloco de código é executado ao menos uma vez, antes da primeira avaliação da condição. A repetição continua enquanto a condição permanecer verdadeira. Observe o exemplo:

    int sensorValue;
do {
  sensorValue = readSensor();  // Lê o valor do sensor
  delay(100);  // Espera 100 milissegundos
} while (sensorValue < 100); // Repete enquanto o valor do sensor for menor que 100

    A estrutura for é projetada para repetir um bloco de código um número predeterminado de vezes. Ela utiliza uma variável de controle que é inicializada, testada em cada iteração e incrementada (ou decrementada) após cada iteração. Essa estrutura é particularmente útil quando o número de repetições é conhecido de antemão. Observe o exemplo:

    for (int i = 0; i < 10; i++) {
  Serial.println(i);  // Imprime o valor de i na porta serial
}

    O operador ternário ? oferece uma forma concisa de expressar uma estrutura if...else simples em uma única linha de código. A expressão antes do ? é avaliada; se for verdadeira, o valor da expressão será o valor após o ?; caso contrário, será o valor após o :. Observe o exemplo:

    int buttonState = digitalRead(buttonPin);
digitalWrite(ledPin, buttonState ? HIGH : LOW); // Liga o LED se o botão estiver pressionado, desliga caso contrário

    Portas de E/S (Entrada/Saída) do Arduino 🔌

    O Arduino possui diversos pinos que permitem interagir com o mundo exterior, lendo dados de sensores (entradas) e controlando atuadores (saídas).

    Pinos Digitais

    São 14 pinos (0 a 13) que podem ser configurados como entrada ou saída de níveis lógicos digitais (HIGH/LOW) usando a função pinMode(). Um pino digital configurado como saída pode fornecer 5V (HIGH) ou 0V (LOW). Quando configurado como entrada, ele pode detectar se a tensão aplicada está próxima de 5V (HIGH) ou 0V (LOW).

    • Pinos 0 e 1: Usados para comunicação serial (TX/RX). Ao usar a comunicação serial, evite usar esses pinos para outras finalidades.
    • Pinos 2 e 3: Podem ser usados para interrupções externas com attachInterrupt(). Interrupções permitem que o Arduino responda a eventos externos de forma rápida e eficiente.
    • Pinos com PWM (3, 5, 6, 9, 10, 11): Podem gerar sinais analógicos simulados usando analogWrite(). PWM (Pulse Width Modulation) é uma técnica para variar a largura de um pulso digital, permitindo controlar a potência fornecida a um dispositivo, como o brilho de um LED ou a velocidade de um motor.
    • AREF: Pino para tensão de referência do conversor A/D. Usado para definir a tensão máxima que será lida pelos pinos analógicos. Por padrão, a tensão de referência é 5V, mas você pode usar uma tensão de referência externa para aumentar a precisão das leituras analógicas.
    • GND: Pino terra (Ground). Usado como referência de tensão zero para o Arduino e circuitos externos.

    Funções para Pinos Digitais:

    As funções a seguir são utilizadas para configurar e interagir com os pinos digitais do Arduino.

    A função pinMode(pino, modo) é utilizada para configurar um pino digital específico (pino) para se comportar como uma entrada ou uma saída. O parâmetro modo define o comportamento do pino e pode ser INPUT, OUTPUT ou INPUT_PULLUP. A configuração como INPUT_PULLUP ativa um resistor interno que mantém o pino em nível lógico alto (HIGH) por padrão. Essa função é geralmente utilizada dentro da função setup() para inicializar a configuração dos pinos. Por exemplo:

    pinMode(2, OUTPUT);   // Configura o pino 2 como saída
pinMode(4, INPUT);    // Configura o pino 4 como entrada
pinMode(7, INPUT_PULLUP); // Configura o pino 7 como entrada com resistor de pull-up

    A função digitalRead(pino) é utilizada para ler o estado lógico (HIGH ou LOW) de um pino digital específico (pino). O pino deve ser previamente configurado como INPUT ou INPUT_PULLUP para que a leitura seja precisa. A função retorna HIGH (1) se a tensão no pino estiver próxima de 5V, e LOW (0) se estiver próxima de 0V. Por exemplo:

    int buttonState = digitalRead(3); // Lê o estado do pino 3 e armazena na variável buttonState

    A função digitalWrite(pino, valor) é utilizada para definir o estado lógico (HIGH ou LOW) de um pino digital específico (pino). O pino deve ser previamente configurado como OUTPUT para que a função possa alterar o estado do pino. O parâmetro valor define o estado do pino e pode ser HIGH (5V) ou LOW (0V). Por exemplo:

    digitalWrite(2, HIGH);  // Define o pino 2 como HIGH (5V)
digitalWrite(5, LOW);   // Define o pino 5 como LOW (0V)

    A função analogWrite(pino, valor) é utilizada para enviar um sinal PWM (Pulse Width Modulation) para um pino PWM específico (pino), simulando uma saída analógica. O pino deve ser um pino PWM (3, 5, 6, 9, 10, 11). O parâmetro valor define a largura do pulso e varia de 0 a 255. O valor 0 desliga o pino (0V), o valor 255 liga o pino completamente (5V), e valores intermediários geram uma tensão proporcional. Não é necessário usar pinMode() para pinos PWM usados com analogWrite(). Por exemplo:

    analogWrite(10, 128); // Envia um sinal PWM com valor 128 para o pino 10 (gera ~2.5V no pino 10)

    A função attachInterrupt(pino_interruptor, ISR, modo) é utilizada para associar uma função (ISR - Interrupt Service Routine) a uma interrupção externa em um pino específico (pino_interruptor). Quando ocorre uma interrupção no pino, a função ISR é executada. O parâmetro pino_interruptor é o número do pino digital (2 ou 3 no Arduino Uno). ISR é o nome da função a ser executada quando a interrupção ocorre. modo define quando a interrupção deve ser disparada e pode ser LOW, CHANGE, RISING ou FALLING. Utilize digitalPinToInterrupt(pin) para converter o número do pino digital para o número de interrupção correto. Por exemplo:

    attachInterrupt(digitalPinToInterrupt(2), buttonISR, RISING); // Associa a função buttonISR à interrupção no pino 2, disparada na borda de subida

    A função detachInterrupt(pino_interruptor) é utilizada para desabilitar a interrupção em um pino específico (pino_interruptor). Após a chamada desta função, a função ISR associada ao pino não será mais executada quando ocorrer uma interrupção. Utilize digitalPinToInterrupt(pin) para converter o número do pino digital para o número de interrupção correto. Por exemplo:

    detachInterrupt(digitalPinToInterrupt(2)); // Desabilita a interrupção no pino 2

    A função pulseIn(pino, valor, espera) é utilizada para medir a duração (em microssegundos) de um pulso (HIGH ou LOW) em um pino específico (pino). O parâmetro valor especifica o estado do pulso a ser medido (HIGH ou LOW). O parâmetro espera (timeout) é opcional e define o tempo máximo de espera em microssegundos. A função retorna a duração do pulso em microssegundos ou 0 se nenhum pulso for detectado dentro do tempo de espera. Por exemplo:

    unsigned long duration = pulseIn(4, HIGH); // Mede a duração de um pulso HIGH no pino 4

    Pinos Analógicos (ANALOG IN)

    São 6 pinos (A0 a A5) usados para ler sinais analógicos (0-5V). Não precisam ser configurados com pinMode() para leitura analógica. O conversor A/D interno tem 10 bits, resultando em valores de 0 a 1023. Podem também ser usados como pinos digitais (pinos 14 a 19). Por exemplo, pinMode(14, OUTPUT) configura A0 como saída digital.

    Função para Pinos Analógicos:

    A função analogRead(pino) é utilizada para ler o valor analógico de um pino específico (pino, que pode ser A0-A5) e retorna um valor inteiro entre 0 e 1023, representando a tensão lida no pino. Por exemplo, int sensor = analogRead(A0); lê o valor analógico do pino A0 e armazena na variável sensor. Se a tensão em A0 for 2.5V, o valor retornado será próximo de 512. Os pinos analógicos A0-A5 também podem ser referenciados como pinos 14-19, portanto analogRead(14) é equivalente a analogRead(A0).

    int potPin = A0;    // Pino analógico conectado ao potenciômetro
int ledPin = 9;     // Pino PWM conectado ao LED

void setup() {
  pinMode(ledPin, OUTPUT);  // Configura o pino do LED como saída
}

void loop() {
  int potValue = analogRead(potPin);           // Lê o valor do potenciômetro (0-1023)
  int ledBrightness = map(potValue, 0, 1023, 0, 255); // Mapeia o valor para a faixa de PWM (0-255)
  analogWrite(ledPin, ledBrightness);           // Define o brilho do LED
  delay(10);                                    // Pequeno delay para estabilizar a leitura
}

    Porta Serial (USB)

    O conector USB permite comunicação serial com um computador e também pode alimentar o Arduino.

    Funções para Porta Serial:

    As funções a seguir são utilizadas para configurar e interagir com a porta serial do Arduino.

    A função Serial.begin(taxa) habilita a comunicação serial e define a taxa de bits por segundo (bps), também conhecida como "baud rate". Essa taxa deve ser a mesma utilizada no Monitor Serial do IDE Arduino para que a comunicação seja estabelecida corretamente. Essa função é geralmente utilizada na função setup(). Por exemplo:

    Serial.begin(9600); // Inicializa a comunicação serial com uma taxa de 9600 bps

    É importante notar que os pinos 0 e 1 (RX e TX) não podem ser utilizados como pinos de entrada/saída digital quando a comunicação serial está habilitada.

    A função Serial.end() desabilita a porta serial, liberando os pinos 0 e 1 (RX e TX) para serem utilizados como pinos de entrada/saída digital. Essa função também é utilizada na função setup(). Por exemplo:

    Serial.end(); // Desabilita a comunicação serial

    A função Serial.available() retorna o número de bytes (caracteres) disponíveis para leitura no buffer serial. Essa função é útil para verificar se há dados disponíveis para serem lidos antes de tentar ler da porta serial. Retorna 0 se não houver dados disponíveis. Por exemplo:

    int bytesAvailable = Serial.available(); // Obtém o número de bytes disponíveis para leitura

    A função Serial.read() lê e retorna o primeiro byte disponível no buffer serial como um valor inteiro. Se não houver dados disponíveis, a função retorna -1. Por exemplo:

    int incomingByte = Serial.read(); // Lê o primeiro byte disponível no buffer serial

    A função Serial.print(valor, formato) envia dados (valor) para a porta serial. O parâmetro formato (opcional) especifica a base numérica para inteiros (BIN para binário, DEC para decimal, HEX para hexadecimal) ou o número de casas decimais para números de ponto flutuante. Por exemplo:

    Serial.print(1.23456, 3);   // Envia "1.234" para a porta serial
Serial.print('A', BIN);    // Envia "01000001" para a porta serial

    A função Serial.println(valor, formato) é similar à função Serial.print(), mas adiciona um caractere de retorno de carro (\r) e uma nova linha (\n) ao final dos dados enviados, movendo o cursor para a próxima linha no Monitor Serial. Por exemplo:

    Serial.println(analogRead(A0)); // Envia o valor lido do pino A0 para a porta serial, seguido de uma nova linha

    Pinos de Alimentação (POWER)

    Localizados geralmente próximos aos pinos analógicos, estes pinos fornecem diferentes opções de alimentação para o Arduino e dispositivos externos.

    • RESET: Conectar este pino ao terra (GND) reinicia o Arduino. É útil para resetar o microcontrolador sem desligar a alimentação.
    • 3V3: Fornece uma saída de 3.3V com corrente máxima de 50mA. Pode ser usada para alimentar sensores e outros dispositivos que operam com essa tensão.
    • 5V: Fornece uma saída de 5V. A corrente máxima disponível depende da fonte de alimentação do Arduino (USB ou Vin).
    • GND: Pinos de terra (Ground). Usados como referência de tensão zero para o Arduino e circuitos externos.
    • Vin: Entrada de tensão. Pode ser usada para alimentar o Arduino com uma fonte de tensão externa não regulada, tipicamente entre 7V e 12V (o datasheet pode especificar entre 9V e 15V, mas 7-12V é mais comum para evitar superaquecimento do regulador). A placa Arduino possui um regulador de tensão que converte essa tensão para 5V e 3.3V.
